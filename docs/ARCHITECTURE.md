## 2. Request Flow

### Data Fetching (Read)
```
User visits page
  → Next.js Server Component renders
  → Drizzle ORM queries MySQL
  → HTML streamed to browser
  → Client Components hydrate
```

### Mutations (Create/Update/Delete)
```
User submits form
  → Client Component calls Server Action
  → Zod validates input
  → Auth.js checks session
  → Drizzle ORM writes to MySQL
  → revalidatePath() or updateTag() refreshes cache
  → UI updates
```

### Authentication Flow
```
User clicks "Sign in with Google"
  → Redirect to Google OAuth consent
  → Google redirects back to /api/auth/callback/google
  → Auth.js creates session in MySQL
  → proxy.ts protects all /dashboard/* routes
  → Session cookie sent with every request
```

## 4. Data Flow Patterns

### Pattern 1: Server Component Data Fetching
```typescript
// In a Server Component (no "use client")
import { db } from "@/db";
import { syllabi } from "@/db/schema";
import { eq } from "drizzle-orm";
import { auth } from "@/lib/auth";

export default async function DashboardPage() {
  const session = await auth();
  const userSyllabi = await db.query.syllabi.findMany({
    where: eq(syllabi.userId, session.user.id),
  });
  return <SyllabusList items={userSyllabi} />;
}
```

### Pattern 2: Server Action with Validation
```typescript
"use server";
import { z } from "zod";
import { db } from "@/db";
import { syllabi } from "@/db/schema";
import { auth } from "@/lib/auth";
import { revalidatePath } from "next/cache";

const CreateSyllabusSchema = z.object({
  courseTitle: z.string().min(1).max(200),
  courseCode: z.string().min(1).max(20),
  // ... etc
});

export async function createSyllabus(input: z.infer<typeof CreateSyllabusSchema>) {
  const session = await auth();
  if (!session) throw new Error("Unauthorized");

  const validated = CreateSyllabusSchema.parse(input); // throws on invalid

  const [syllabus] = await db.insert(syllabi).values({
    ...validated,
    userId: session.user.id,
  }).$returningId();

  revalidatePath("/dashboard");
  return syllabus;
}
```

### Pattern 3: Client Component with Server Action
```typescript
"use client";
import { createSyllabus } from "@/actions/syllabus";
import { useTransition } from "react";

export function SyllabusForm() {
  const [isPending, startTransition] = useTransition();

  function onSubmit(formData: FormData) {
    startTransition(async () => {
      await createSyllabus({
        courseTitle: formData.get("courseTitle"),
        courseCode: formData.get("courseCode"),
      });
    });
  }

  return <form action={onSubmit}>...</form>;
}
```

## 5. Authentication Architecture

### Auth.js v5 + Google OAuth + Drizzle Adapter

```
Google OAuth Provider
    ↓
Auth.js v5 (src/lib/auth.ts)
    ↓
Drizzle Adapter → MySQL (users, accounts, sessions tables)
    ↓
Session cookie (encrypted JWT)
    ↓
proxy.ts validates cookie on every protected route
```

### Tables Required for Auth
- `users` — id, name, email, emailVerified, image
- `accounts` — OAuth provider linkage (Google)
- `sessions` — Session tokens
- `verificationtokens` — Email verification (if needed)

These are defined in `src/db/schema.ts` alongside the syllabus tables.

## 7. Caching Strategy (Next.js 16)

Next.js 16 uses **explicit caching** (Cache Components). By default, all dynamic code runs at request time.

- **Server Components**: Data fetched fresh per request (no implicit caching)
- **`"use cache"` directive**: Opt-in caching for specific components/functions
- **`revalidatePath()`**: After Server Action mutations, revalidate affected paths
- **`updateTag()`**: For read-your-writes semantics in Server Actions

For this application (100 users, data changes frequently), we default to **no caching** and use `revalidatePath()` after mutations.

## 8. Error Handling Strategy

### Server Actions
```typescript
// Always return a typed result, never throw to client
type ActionResult<T = void> =
  | { success: true; data: T }
  | { success: false; error: string };

// Example:
export async function createSyllabus(input): Promise<ActionResult<Syllabus>> {
  try {
    // ... validation + DB
    return { success: true, data: syllabus };
  } catch (e) {
    if (e instanceof ZodError) {
      return { success: false, error: e.errors[0].message };
    }
    return { success: false, error: "Something went wrong" };
  }
}
```

### Error Boundaries
- `src/app/(dashboard)/error.tsx` — Catches route-level errors
- `src/app/(dashboard)/not-found.tsx` — 404 page
- `src/app/global-error.tsx` — Catches root layout errors
