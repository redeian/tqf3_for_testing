# Superpowers Integration

> **For AI Agents and Humans:** This document defines how the Superpowers skill framework integrates with our development process. Read this before starting any implementation work.

## What Superpowers Is

Superpowers is an agentic skills framework that gives coding agents a structured software development methodology. Instead of jumping straight into code, the agent:

1. **Brainstorms** — asks clarifying questions, proposes designs, gets approval
2. **Plans** — breaks work into bite-sized tasks (2-5 min each) with exact code and TDD steps
3. **Executes** — implements tasks one by one with subagent-driven development
4. **Tests** — enforces strict RED-GREEN-REFACTOR TDD
5. **Reviews** — two-stage code review (spec compliance, then code quality)
6. **Finishes** — presents options to merge, create PR, keep, or discard

## How It Integrates With Our Process

Our 6-phase development process (see [`docs/DEVELOPMENT_PROCESS.md`](../DEVELOPMENT_PROCESS.md)) maps to Superpowers as follows:

```
Our Process                          Superpowers Skill
──────────────────────────────────────────────────────────
Phase 1: Discovery                   (Brainstorming — partially)
  You interview users
  AI agent structures PRD

Phase 2: Diagrams                    (Brainstorming — design)
  AI agent creates diagrams
  You validate with users            ← Human Checkpoint HC-2

Phase 3: Architecture                (Writing Plans)
  AI agent designs schema/auth
  AI agent writes implementation plan ← Plan saved to docs/superpowers/plans/
  You review schema                  ← Human Checkpoint HC-3

Phase 4: Coding                      (Subagent-Driven Development + TDD)
  AI agent executes plan task-by-task
  Each task: RED → GREEN → REFACTOR → commit
  Two-stage review per task
  You review each PR                 ← Human Checkpoint HC-4

Phase 5: Deployment                  (Finishing a Development Branch)
  You merge to main
  CI/CD auto-deploys

Phase 6: Feedback Loop               (Brainstorming → Planning → Execute)
  New requirement → back to Phase 1-4
```

## Plan File Format

Every plan generated by Superpowers must follow this header:

```markdown
# [Feature Name] Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** [One sentence describing what this builds]

**Architecture:** [2-3 sentences about approach]

**Tech Stack:** Next.js 16, Drizzle ORM, Auth.js v5, Zod, Vitest, Playwright

**Source Documents:**
- PRD: `docs/syllabus_system_prd.md`
- Architecture: `docs/ARCHITECTURE.md`
- Coding Standards: `docs/CODING_STANDARDS.md`
- Testing: `docs/TESTING.md`
- Error Handling: `docs/ERROR_HANDLING.md`
- Relevant diagrams: `docs/diagrams/activity/<feature>.md`, `docs/diagrams/sequence/<feature>.md`

---
```

## Task Execution Record Format

After each task is implemented, the AI agent creates a record in `docs/superpowers/executions/`:

```markdown
# Task [N]: [Task Name] — Execution Record

**Date:** YYYY-MM-DD
**Plan:** docs/superpowers/plans/YYYY-MM-DD-<feature-name>.md
**Task:** Task N
**Status:** ✅ Complete / ❌ Blocked / 🔄 In Progress

## What Was Implemented

- Created: `src/path/to/file.ts`
- Modified: `src/path/to/existing.ts`
- Tests: `tests/unit/path/to/test.ts`

## TDD Cycle

### RED — Failing Test
- Test: `test('should ...')`
- Command: `npm test -- --run tests/unit/path/to/test.ts`
- Result: FAIL — "function not defined" ✅ (expected failure)

### GREEN — Minimal Implementation
- Code written: [brief description]
- Command: `npm test -- --run tests/unit/path/to/test.ts`
- Result: PASS ✅

### REFACTOR
- Changes: [what was refactored, if any]
- All tests still pass: ✅

## Code Review

### Stage 1: Spec Compliance
- Reviewer: Subagent
- Result: ✅ All requirements met / ❌ Issues found
- Issues (if any): [list]

### Stage 2: Code Quality
- Reviewer: Subagent
- Result: ✅ Clean / ❌ Issues found
- Issues (if any): [list]

## Commit
```
git commit -m "feat(scope): description"
SHA: abc1234
```

## Human Verification
- [ ] Human reviewed this task
- [ ] Tests pass locally
- [ ] Code matches the plan
- [ ] Ready to proceed to next task

**Human sign-off:** [name] on [date] — ✅ Approved / ❌ Changes requested
```

## The Complete Flow (With Recording)

```
1. User gives requirement / PRD is ready
   ↓
2. AI Agent reads PRD + Architecture + Diagrams
   ↓
3. AI Agent uses Superpowers "writing-plans" skill
   → Creates plan: docs/superpowers/plans/YYYY-MM-DD-<feature>.md
   → Plan contains N tasks, each with TDD steps, exact code, commit instructions
   ↓
4. Human reviews the plan
   → Creates: docs/superpowers/verifications/YYYY-MM-DD-<feature>/plan-review.md
   → ✅ Approved or ❌ Changes requested
   ↓
5. AI Agent uses Superpowers "subagent-driven-development" skill
   → For EACH task:
     a. Dispatch implementer subagent (writes failing test → implements → passes test)
     b. Dispatch spec reviewer subagent (checks code matches plan)
     c. Dispatch code quality reviewer subagent (checks code quality)
     d. Create execution record: docs/superpowers/executions/YYYY-MM-DD-<feature>/task-N-record.md
     e. Commit with conventional commit message
   ↓
6. After each task (or batch of tasks):
   → Human reviews the execution record(s)
   → Creates: docs/superpowers/verifications/YYYY-MM-DD-<feature>/phase-4-task-N.md
   → ✅ Approved → proceed to next task
   → ❌ Changes requested → AI agent fixes → re-verify
   ↓
7. All tasks complete
   → AI Agent uses Superpowers "finishing-a-development-branch" skill
   → Human chooses: Merge / PR / Keep / Discard
   → Creates: docs/superpowers/verifications/YYYY-MM-DD-<feature>/phase-4-complete.md
   ↓
8. Merge to main → CI/CD deploys → Post-deploy health check
   → Creates: docs/superpowers/verifications/YYYY-MM-DD-<feature>/deployment.md
```

## Recovery Process

If something goes wrong mid-implementation:

### Scenario: AI agent made a mistake in Task 5
```
1. Check the execution record: docs/superpowers/executions/<feature>/task-05-record.md
2. See what was done, what tests passed, what review found
3. Identify the mistake
4. Either:
   a. Tell AI agent to fix Task 5 → new commit → update execution record
   b. Git revert Task 5's commit → re-execute from plan
```

### Scenario: Need to pause and resume later
```
1. All plans are in: docs/superpowers/plans/
2. All execution records are in: docs/superpowers/executions/
3. All verification logs are in: docs/superpowers/verifications/
4. To resume: read the plan + last execution record + last verification
5. Continue from the next unverified task
```

### Scenario: Need to audit what happened
```
1. Read the plan: docs/superpowers/plans/<feature>.md
   → What was supposed to happen
2. Read execution records: docs/superpowers/executions/<feature>/
   → What actually happened (task by task)
3. Read verification logs: docs/superpowers/verifications/<feature>/
   → What humans approved and when
4. Git log: git log --oneline --grep="<feature>"
   → All commits related to this feature
5. Full audit trail: plan → executions → verifications → git history
```

## AI Agent Rules for Superpowers

1. **Always save plans** to `docs/superpowers/plans/` with date prefix
2. **Always create execution records** after each task in `docs/superpowers/executions/`
3. **Always wait for human verification** before proceeding to the next task (when using executing-plans) or next batch (when using subagent-driven-development with checkpoints)
4. **Never delete** plan or execution files — they are the audit trail
5. **Reference source documents** in every plan header (PRD, Architecture, etc.)
6. **Follow TDD strictly** — RED (failing test) → GREEN (minimal code) → REFACTOR → commit
7. **Use conventional commits** — `feat(scope): description`
8. **If blocked, stop and ask** — don't guess or skip steps
