# 📚 Technical Documentation Index

> **For AI Agents:** This is your entry point. Read this file first to understand what documentation exists and where to find it.

## Three-Stage Architecture

This boilerplate has three independent stages. Each has its own documents:

| Stage | Purpose | Key Documents | Output |
|-------|---------|---------------|--------|
| **1. Design** | Gather requirements, create diagrams, define architecture | PRD, Diagrams, ER Schema, Design System | Design Package (versioned) |
| **2. Implementation** | Code the system task-by-task with TDD + review | Architecture, Coding Standards, Superpowers Plans | Working code + tests |
| **3. Verification** | Test, verify against design, deploy to production | Testing Strategy, Deployment Guide, Health Checks | Live system |

## How AI Agents Should Use These Docs

1. **Always start here** — `docs/INDEX.md` (this file)
2. **Read AGENTS.md** — Project-specific rules for AI agents
3. **Read DEVELOPMENT_PROCESS.md** — Understand the 6-phase lifecycle and human checkpoints
4. **Before planning a feature** → Read the PRD + relevant activity/sequence diagrams
5. **Before writing code** → Read CODING_STANDARDS.md + ARCHITECTURE.md
6. **Before writing database code** → Read database_schema.md + diagrams/er.md
7. **Before writing UI** → Read Design System + look at UI mockup
8. **Before deploying** → Read DEPLOYMENT.md
9. **After making changes** → Run tests (`npm test` and `npm run test:e2e`)
10. **After any scope change** → Update PRD, diagrams, and CHANGELOG.md

## Document Maintenance Rules

- **All docs live in `docs/`** — never store technical docs outside this folder
- **Use relative paths** in all cross-references (e.g., `./database_schema.md`, not `file:///Users/...`)
- **Keep docs in sync** — if you change the schema, update `database_schema.md`; if you change the stack, update the PRD
- **No external dependencies** — docs must be fully self-contained within the project repo
- **Commit docs with code** — documentation changes go in the same PR as code changes
